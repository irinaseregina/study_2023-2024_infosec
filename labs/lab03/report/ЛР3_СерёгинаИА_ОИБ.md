# Цель работы

Получение практических навыков работы в консоли с атрибутами файлов для групп пользователей.

# Задание

1. Создать нового пользователя и добавить его в группу.
2. Заполнить таблицы 3.1 и 3.2.


# Выполнение лабораторной работы

1. Создаю нового пользователя guest2, задаю для него пароль и добавляю его в группу guest (рис. [-@fig:001]).

![новый пользователь](image/1.jpg){#fig:001 width=70%}

2. Захожу под имененм нового пользователя и проверяю путь директории (рис. [-@fig:002]).

![путь директории](image/2.jpg){#fig:002 width=70%}

3. Теперь проверяю путь директории для пользователя guest (рис. [-@fig:003]).

![путь директории](image/3.jpg){#fig:003 width=70%}

4. Используя команды whoami, id, groups, id -Gn (названия групп пользователя), id -G (коды групп пользователя) проверяю информацию о пользователе guest (рис. [-@fig:004]).

![информация о пользователе guest](image/4.jpg){#fig:004 width=70%}

5. Используя те же команды, вывожу информацию о пользователе guest2 (рис. [-@fig:005]).

![информация о пользователе guest2](image/5.jpg){#fig:005 width=70%}

6. Использую etc/group, вывожу пользователей групп (рис. [-@fig:006]).

![файл etc/group](image/6.jpg){#fig:006 width=70%}

7. Регистрирую guest2 в группе guest, использую newgrp (рис. [-@fig:007]).

![регистрирую пользователя в группе](image/7.jpg){#fig:007 width=70%}

8. Добавляю права пользователям группы guest. Права на чтение, запись и исполнение для директории home/guest. Снимаю атрибуты с dir1 (рис. [-@fig:008]).

![работаю с правами и атрибутами](image/8.jpg){#fig:008 width=70%}

9. Проверяю возможность guest2 работать с файлами dir1 (рис. [-@fig:009]).

![проверка возможностей guest2](image/9.jpg){#fig:009 width=70%}
